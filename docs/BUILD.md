# DeCoupled-AI Build Guide

This guide covers compiling the DeCoupled-AI native systems engine across different platforms and hardware accelerators.

## Prerequisites

- **Rust**: Latest stable version (1.75+)
- **LLVM/Clang**: Required for various backend bindings.

### Hardware-Specific Requirements

- **CUDA (NVIDIA)**:
  - CUDA Toolkit 11.8+
  - `nvcc` must be in your PATH.
  - Environment variable `CUDA_PATH` or `CUDA_HOME` should be set.
- **ROCm (AMD)**:
  - ROCm 5.0+ installed on Linux.
  - `hipcc` must be in your PATH.
- **Metal (Apple)**:
  - macOS with Xcode Command Line Tools.
- **CPU (Fallback)**:
  - No special hardware required. Supports AVX2, AVX-512, and NEON via runtime detection.

## Build Commands

DeCoupled-AI uses Cargo features to manage backend selection.

### 1. Default (CPU Only)
```bash
cargo build --release
```

### 2. NVIDIA CUDA
```bash
cargo build --release --features cuda
```
*Note: This will trigger `nvcc` compilation of kernels to PTX.*

### 3. Apple Metal
```bash
cargo build --release --features metal
```

### 4. AMD ROCm
```bash
cargo build --release --features rocm
```

## Platform-Specific Notes

### Windows
- For **CUDA**, ensure `cl.exe` (from MSVC) is in your path so `nvcc` can find it.
- **mmap** uses the Win32 `CreateFileMapping` API via the `mem-windows` crate.

### Linux
- **Huge Pages**: To enable huge page support for weights, run:
  ```bash
  echo 1024 | sudo tee /proc/sys/vm/nr_hugepages
  ```
- **Shared Memory**: The engine can use `shm_open` for zero-copy cross-process inference.

### macOS
- **Metal** performance is optimized via a buffer pool and pre-compiled shading language kernels.

## Troubleshooting

- **NVCC Not Found**: Ensure CUDA Toolkit is installed and `CUDA_PATH` is set.
- **Permission Denied (Huge Pages)**: Running without `sudo` may fallback to standard 4KB pages.
- **Linking Errors**: If `cublas` or `cudart` cannot be found, check `cargo:rustc-link-search` paths in `compute-cuda/build.rs`.
